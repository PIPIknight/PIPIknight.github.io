自动登录与无感刷新 #

​ 把这两种功能放在一块儿是因为，这两种功能都是围绕 通过 refreshToken 获取新的 token 展开的！

自动登录 #

​ 首先讨论自动登录，refreshToken存于Cookie中，当token清除而refreshToken依旧存在时，说明当前用户可以自动登录。通过发送刷新token请求获取新的token。

​ 首先要搞清楚要把 自动登录 放置在那里，我认为在两个地方，即路由前置拦截以及main.js主入口文件当中。

​ 首先，放置于main.js主入口文件中很容易理解，访问页面必然免不了执行。在最初就判断其登录状态进行自动登录，以下是代码：

js
// 摘自我的一个其中一个项目
import { createApp } from 'vue'
……
// 刷新 token 功能
import { refreshFunc } from "@/utils/axios/refreshToken";
……

const app = createApp(App)

// 刷新 + 7天免登录
(async function init() {
    NextLoading.start()
    let token = getToken()
    let refreshToken = getRefreshToken()
    if (!token && refreshToken) {
        // 存在 refreshToken 却不存在 token 时，获取新token自动登录
        try {
            const { data: res } = await refreshFunc()
            token = res.data.accessToken
            // 保存 token
            setToken(token, res.data.expiresIn)
        } catch (err) {
            ElMessage({
                message: 'Token刷新失败,请重新登录!',
                type: 'error',
                duration: 5000
            })
        }
    }
    // 这里的作用时，用token去获取用户登录状态必要的信息，也是防止刷新丢失 vuex等内存数据丢失！
    if (token) {
        await useUserInfo().setUserInfos()
    }
})().finally(() => {
    // 一切准备就绪再挂在app
    NextLoading.done()
    app.use(pinia).use(router).use(ElementPlus, { i18n: i18n.global.t }).use(i18n).mount('#app')
})

​ 由代码可见，挂在app的行为放置在了登录及获取必要的登录状态数据后，这样做的好处是model先于view，附带的好处便是刷新后内存丢失的数据重新获取。

​ 其次，光在main.js放置自动登录还不够，在路由前置中也需要放置一个自动登录，防止的是 在访问途中token过期从而客户被自动登出 ！代码如下：

js
router.beforeEach(async (to, from, next) => {
    NProgress.configure({ showSpinner: false });
    if (to.meta.title) NProgress.start();
    let token = getToken();
    let refreshToken = getRefreshToken()
    if (!token && refreshToken) {
        // 当只有refreshToken时，可能token已经过期(该项目token有效时间2小时！)，去刷新token
        try {
            const { data: res } = await refreshFunc()
            token = res.data.accessToken
            setToken(token, res.data.expiresIn)
        } catch (err) {
            ElMessage({
                message: 'Token刷新失败,请重新登录!',
                type: 'error',
                duration: 5000
            })
        }
        // 这里放置刷新userInfo的位置与main.js存在区别！想想看为什么呢？？
        if (token) {
            await useUserInfo().setUserInfos()
        }
    }

无感刷新 #

​ 无感刷新，网上能百度到的方案一大把，这里简要说一下。

​ 通常一个项目都会设置一个 axios 实例，然后对其设置统一的请求拦截和相应拦截处理。首先明确一点，无感刷新是设置在 响应拦截的失败情况 下的。其次，并不是所有的失败都需要进行无感刷新，一般只有 401（ 未授权 ） 和 403（ 服务器拒绝执行 ）时，进行无感刷新。所以需要对错误进行判断。最后，最重要的一点，刷新token的请求时一个单独的axios实例，有别于通用的那个axios实例。以下是代码：

js
import axios from "axios"
import { getRefreshToken } from "../storage"

export const refreshFunc = () => {
    // 一个单独的axios实例
    return axios({
        url: '/api/user/v1/account/refresh', // 刷新请求接口
        method: 'post',
        baseURL: import.meta.env.VITE_API_URL as any,
        timeout: 50000,
        data: {
            refreshToken: getRefreshToken()
        }
    })
}


// 通用实例的响应拦截
// 添加响应拦截器
service.interceptors.response.use(
    // 正常情况不做处理
    (response) => {
        const res = response.data;
        if (res.code && res.code !== 0) {
            return Promise.reject(res);
        } else {
            return response.data
        }
    },
    (error) => {
        // 对响应错误做点什么
        if (error.message.indexOf('timeout') != -1) {
            ElMessage.error('网络超时');
        } else if (error.message == 'Network Error') {
            ElMessage.error('网络连接错误');
        } else {
            if (error.response.data) ElMessage.error(error.response.statusText);
            else ElMessage.error('接口路径找不到');
        }
        // 当状态是401或者403时，推测可能是token过期，进行一次无感刷新
        if (error.response.status === 401 || error.response.status === 403) {
            // 先判断有没有refreshToken,没有refreshToken时
            if (!getRefreshToken()) {
                // 没有且当前页面不在白名单，强制退出
                if (!whiteList.includes(router.currentRoute.value.path)) {
                    return router.push({ path: '/login' })
                }
                return
            }
            // 保存该次请求的config配置
            const requestConfig = error.response.config
            // 获取到了refreshToken，请求刷新
            refreshFunc()
                .then((response: any) => {
                    const { data: res } = response
                    if (res.code === 0) {
                        const { token, expiresIn } = res.data
                        setToken(token, expiresIn)
                        // 重置该次请求的token
                        requestConfig.headers.Authorization = 'Bearer ' + token
                        // 重发请求
                        return service.request(requestConfig)
                    } else {
                        // 刷新失败，应该清除 refreshToken 然后退出！
                        failToRefresh()
                        return

                    }
                })
                .catch(() => {
                    failToRefresh()
                    return
                })
        }
        return Promise.reject(error);
    }
);

​ 由于这个项目不是传统的后台管理项目，而是云平台项目，一些处理仍旧比较粗糙有待改进。但是大致内容也表达得清楚了。
