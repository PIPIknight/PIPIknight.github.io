图片img和base64格式的相互转换 #

​ 业务场景是，el-upload上加载营业执照，并展示缩略图。点击上传手动上传。老大要求图片以base64而不是formData的形式上传。

​ 如果，客户曾经上传过营业执照，则当登陆时，el-upload展示已上传的图片缩略图。若未改变现有的营业执照图片，则返回原上传文件。

​ 涉及到的就是一个图片转base64互相转换的过程，多次遇到过却每次都要百度太不像话了，这里记录一下。

​ 图片和base64互相转换主要用到 FileReader 类，MDN对其描述：

​ https://developer.mozilla.org/zh-CN/docs/Web/API/FileReader

​ FileReader 对象允许 Web 应用程序 异步读取 存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用 File 或 Blob 对象指定要读取的文件或数据。

​ 其中 File 对象可以是来自用户在一个元素上选择文件后返回的FileList对象，也可以来自拖放操作生成的 DataTransfer对象，还可以是来自在一个HTMLCanvasElement上执行mozGetAsFile()方法后返回结果。

​ 先上相关代码：

img->base64 #

​

ts
// 校验上传图片规范
const validateSelected = async (uploadFile: any) => {
    // 校验大小
    if (uploadFile.size / 1024 / 1024 > 10) {
        ElMessage.error('图片大小不能超过10M')
        // isFailed.value = true
        return
    }
    // 上传的文件预览图用 URL.createObjectURL 创建一个 DOMString，其中包含一个表示参数中给出的对象的 URL。这个 URL 的生命周期和创建它的窗口中的 document 绑定。
    imageUrl.value = URL.createObjectURL(uploadFile.raw)
    // 这里强调 FileReader 是异步读取的
    busiLicense.value = await getBase64(uploadFile.raw)
}	

// File转base64
const getBase64 = (file: any) => {
    return new Promise(function (resolve, reject) {
        if (!file) {
            resolve('')
        }
        let reader = new FileReader()
        let imgResult: string | ArrayBuffer | null
        // readAsDataURL 方法会读取指定的 Blob 或 File 对象。读取操作完成的时候，readyState 会变成已完成DONE，并触发 loadend (en-US) 事件，同时 result 属性将包含一个data:URL 格式的字符串（base64 编码）以表示所读取文件的内容。
        reader.readAsDataURL(file)
        // FileReader onload事件在读取操作完成时触发
        reader.onload = function () {
            imgResult = reader.result
        }
        reader.onerror = function (error) {
            // 失败会提前结束 Promise，虽然 onloadend 依旧会触发，但是Promise已转为fulfilled
            reject(error)
        }
        // FileReader onloadend事件在读取操作结束（无论成功失败）发生，在onload之后发生
        reader.onloadend = function () {
            resolve(imgResult)
        }
    })
}

步骤相当明确：

	1. 获得 input 选中的文件内容 
	2. 创建一个 Promise 并新建 FileReader 对象，用 FileReader 的事件结果作为 Promise 的处理结果
	3. FileReader调用**readAsDataURL**方法处理文件数据获得对应**result**，即base64字符串

值得注意的是，调用不同的方法，FileReader最终得到的result的结果也不同：

​ readAsDataURL() ==> result "data:[file]/[type];base64,……(base64 编码内容)"

​ readAsArrayBuffer() ==> rseult ArrayBuffer

​ readAsText() ==> 必须要挂载 实例下的 onload 或 onloadend 的方法处理转化后的结果( 字符串形式 )

base64->img #

​ 后台返回 base64 字符串，不能直接 FileReader进行处理，而是先需要文件的转化以及 Uint8Array 文件用于 Blob 转换，获得可下载的原文件

​

js
// 获取营业执照图片
const getCertifImg = async () => {
    const params = {
		……
    }
    // 从后端获取到了上传文件的文件流
    const { data } = await user.getImg(params)
    const reader = new FileReader()
    // 原有执照预览！
    reader.onload = function (e: any) {
        // 获取图片预览
        imageUrl.value = e.target.result
    }
    // ———————————————————————————— 转换开始 ————————————————————————————
    // 去除掉所有字符串中的空白符
    let base64Data = data.replace(/[\n\r]/g, '')
    // window.atob 对经过 base-64 编码的字符串进行解码（window.atob则是编码）
    let raw = window.atob(base64Data)
    let n = raw.length
    // 新建一个 8 位无符号整型数组
    let u8arr = new Uint8Array(n)
    while (n--) {
        // 三元组表达像素颜色
        u8arr[n] = raw.charCodeAt(n)
    }
    const fileType = …… // 图片包括 png,jpg,jpeg等，这个由另一个后端接口返回
    
    // Blob() 构造函数返回一个新的 Blob 对象。blob 的内容由参数数组中给出的值的串联组成。
    reader.readAsDataURL(new Blob([u8arr], { type: 'image/' + fileType }))
    
    busiLicense.value = `data:image/${fileType};base64,` + data
    // 下载文件，验证是否与原文件相同
    // const a = document.createElement('a')
    // a.href = `data:image/${fileType};base64,` + data
    // a.setAttribute('download', 'license')
    // a.click()
    // document.body.removeChild(a)
}

​ 上述代码中， FileReader更多的作用是生成预览图，正如 FileReader 的描述：

​ FileReader 对象允许 Web 应用程序 异步读取 存储在用户计算机上的文件（或原始数据缓冲区）的内容（获取内容），使用 File 或 Blob 对象指定要读取的文件或数据**（读取内容）**。

​ 如果说，img->base64我用FileReader获取输入的结果，这里我们实现的是 供给 读取的内容。
