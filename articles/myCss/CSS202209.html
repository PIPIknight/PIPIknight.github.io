CSS 9月部分总结 #

对img标签使用before和after伪元素失效 #

​ 在W3C中对伪元素 before 和 after 中的定义：

As their names indicate, the :before and :after pseudo-elements specify the location of content before and after an element's document tree content.

https://www.w3.org/TR/CSS21/generate.html#before-after-content

是一个替换元素，其外观和尺寸是由外部资源来决定的，当外部图片资源加载失败时其会显示破裂图片和alt文字，尺寸仅由其自身内容决定。这时元素可以使用伪元素:before和:after，因为其元素内容没有被替换！

Note. This specification does not fully define the interaction of :before and :after with replaced elements (such as IMG in HTML). This will be defined in more detail in a future specification.

综上，对于可替换元素（如 img、input、select 等）尽量避免在其上使用伪元素

对于svg，或者img图片的异色处理 filter : drop-shadwo() #

css
filter : drop-shadow(offset-x offset-y blur-radius spread-radius color)

MDN上对其描述：

​ 这个函数有点类似于 box-shadow 属性。box-shadow 属性在元素的整个框后面创建一个矩形阴影，而 drop-shadow() 过滤器则是创建一个符合图像本身形状 (alpha 通道) 的阴影。

img意外撑开div的问题 #

​ 在独立的img元素套用div作为父级元素时，常规思维认为div边框=img边框，但是却意外会发现div边框多了4px左右的像素。

​ 原因：img属于行内元素，在结束的时候，会在末尾加上一个空白符，而空白符默认会有行高的，所以会自动将div撑开4px。

​ 解决办法：display : inline-block，ps：img属于行内元素！

更改元素变形的原点 transform-origin #

一个值：

​ 必须是 长度，百分比，或 **left, center, right, top, bottom **关键字中的一个。

两个值：

​ 其中一个必须是 长度，百分比，或 left, center, right 关键字中的一个。 ​ 另一个必须是 长度，百分比，或 top, center, bottom 关键字中的一个。

三个值：

​ 前两个值和只有两个值时的用法相同。 ​ 第三个值必须是 长度。它始终代表 Z 轴偏移量。

css
transform-origin: center; // 中心点
transform-origin: top left; // 左上角
transform-origin: 50px 50px; // 左上角为原点 向下/向右偏移50px
transform-origin: bottom right 60px; //  右下角为原点， 60px代表 Z 轴偏移量

结合 transform : rotate() 实现元素的旋转动画

在el-popper上改样式 #

​ 由于业务需求，要在el-popper上弹出的内容修改样式。直接在当前vue文件中做样式穿透是不会生效的！

需要在主入口文件引入样式时预设好，建议新建一个custom.css文件，设置所有的特殊弹出框样式！
