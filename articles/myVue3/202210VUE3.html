reactive数据重新赋值怎么办 #

​ 一个变量刷新，一个个赋值还不如重新赋值来得方便，但是这个变量是reactive响应式的，一旦赋值将失去响应！（ Proxy变为普通的Object ）。

  1. 利用ref代替reactive响应式

    ​

    js
    let targetObj = ref({……})
targetObj.value = {...newObj}

    ​ 显然你要这么做，那么响应式绑定的便是 targetObj 本身，每次更新都要重新赋值。所有，对于那些数据量大且一次更新内容量大，或者更新频率低的数据，用 ref 响应式绑定并赋值更新是可取且便捷的。

  2. reactive声明包裹到对象中

    ​

    js
    const data = reactive({ targetObj:{……} })
data.targetObj = {...newObj}

    ​ 依旧是 reactive 的响应式，这样做的好处是，reactive是支持深层响应的，支持单个属性频繁变动，直接 data.targetObj.xxx = xxx 就可以了。但是尽量避免多层嵌套的数据造，这样处理的数据最好是扁平化的！

img的动态src错误/丢失 #

​ 官网中其实详细描述了其解决办法：

// 获取assets静态资源，
export const getAssetsFile = (url: string) => {
    return new URL(`../assets/${url}`, import.meta.url).href
}

这里解释一下 URL() 和 import.meta

URL #

​ URL() 构造函数返回一个新创建的 URL 对象，表示由一组参数定义的 URL。

js
const url = new URL(url [, base])

1. url：是一个表示绝对或相对 URL 的 DOMString。如果url 是相对 URL，则会将 base 用作基准 URL。如果 url 是绝对 URL，则无论参数base是否存在，都将被忽略。
2. base：是一个表示基准 URL 的 DOMString，在 url 是相对 URL 时，它才会起效。如果未指定，则默认为 ''。

URL 接口的 href 属性是一个包含完整 URL 的 USVString 值。

import.meta #

​ import.meta是一个给 JavaScript 模块暴露特定上下文的元数据属性的对象。它包含了这个模块的信息，比如说这个模块的 URL。

​ 在一个 ES 模块运行的时候，import.meta 就会存在的一个内含各种有用元数据的对象。而 import.meta.url 包含了对于目前 ES 模块的绝对 URL 路径。

特效：向下展开 #

这个特效看似和上面的很相似，其实完全不同。

​ 上面特效师控制图像从 下部 40px 中出现，超出部分用外部div的 overflow:hidden 处理

​ 这个特效是直接控制显示div的，y的高度 height，并控制展开原点 transform-origin: top

​ 另外，vue中的hover事件需要通过事件模拟实现！ 这里采用了 mouseover 和 mouseleave 模拟hover

vue
<template>
	<div class="nav-item show-production" @mouseover="triggerEnter" @mouseleave="triggerLeave">
        <span>产品列表</span>
        <SvgIcon name='icon-arrow_down.svg' class="svg-icon" :size='16' color='' />
    </div>
	<transition name="plane-anime">
        <div v-show="showProduction" ref="productionPlane" class="sub-nav-plane production-plane"
             @mouseover="triggerEnter('production')" @mouseleave="triggerLeave('production')">
            产品列表展示
    	</div>
    </transition>
</template>
<script lang="ts">
import { ref, reactive, computed, onMounted, defineComponent, toRef, watch } from 'vue'
export default defineComponent({
    setup() {
        let showProduction = ref(false)
        let productionT: any
        const triggerEnter = () => {
            // 清除鼠标移除事件
            clearTimeout(productionT)
            showProduction.value = true
        }
        const triggerLeave = () => {
            // 因为鼠标由导航栏移动到内容plane，会导致内容plane收起，用延时可以有效处理这个问题
            productionT = setTimeout(() => {
                showProduction.value = false
            }, 150)
        }
        return {
            avatarPath,
            avatarMask,
            showChange
        }
    }
}) 
</script>

vue-router 的 addRoutes 不生效的原因 #

​ 帮助其他朋友解决问题，记录一下

主要几点：

​ 将addRoutes触发的时机放在了路由全局守卫beforeEach中了。因为 addRoutes 是路由访问之前的钩子，当触发 beforeEach实际已经处于访问状态中了，当然不生效。这里建议，放置在main.js中，在整体初始化时添加路由！

​ （这是我同事遇到的情况）addRoutes添加的路由在 this.$router.option里看不到。其实动态路由通过 this.$router.options 找不到，这是 vue-router 故意这么设计的。以下是作者的回复

https://github.com/vuejs/vue-router/issues/1859

​ options is the object passed to the vuerouter constructor. It's not modified afterwards.

所以直接用vuex啦！

​ 当然vue3根本不会遇到上述情况，this没有，$无用，vuex被pinia取代
